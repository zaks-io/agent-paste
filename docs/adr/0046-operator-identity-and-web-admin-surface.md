# Operator Identity and Web Admin Surface

Status: Accepted. Refines [ADR 0040](./0040-platform-lockdown-for-operator-initiated-takedown.md)'s deferred operator-interface decision.

Operator-only actions — **Platform Lockdown** changes, on-demand storage-key rotation, abuse review — run through admin routes on the `api` Worker, served behind a web admin section of `app.agent-paste.sh`. Production admin routes are edge-gated by Cloudflare Access and then application-gated by `requireOperator()`. There is no separate operator service. The **Operator** identity is a **Workspace Member** Auth0 session whose authenticated email is on the `OPERATOR_EMAILS` Worker secret allowlist; the `requireOperator()` middleware enforces the check after Auth0 resolves the session and rejects every non-matching session with the generic `not_found` envelope from ADR 0036. **API Key** authentication is rejected outright on admin routes. The CLI in `apps/cli` does not expose any operator-only command.

## Considered Options

- **Wrangler-only local operator interface.** No HTTP admin surface; every operator action is a `wrangler` script run from a local environment with deploy credentials. Maximally secure (no public attack surface) but unworkable: the operator must reach the platform from a phone or a borrowed machine during an incident, and there is no shared admin UI for review surfaces (abuse queue, lockdown listing). Rejected: contradicts the user's explicit requirement to control the platform from the web.
- **Auth0 plus email allowlist only.** Simple and still server-enforced, but platform-wide actions deserve an edge gate as well as app auth. Rejected for production; acceptable only for local development and preview environments.
- **Admin endpoints on the public api, gated only by the `requireScopes` middleware.** Reuses the existing **Scope** machinery by inventing operator-grade **Scopes** like `platform:lockdown:write`. The risk is that **Scopes** are designed to be granted to **API Keys**; once an operator-grade **Scope** exists, an accidental grant on an admin-owned **API Key** would punch a hole in the platform boundary. Rejected: blurs the boundary between workspace authorization and platform authorization.
- **Separate `admin` Worker on its own subdomain.** Cleaner isolation; the admin Worker would bind only the resources it needs and could enforce a different CSP and auth chain. The cost is real: another Worker to deploy, monitor, version, and keep in sync with `api` schema changes, and another Hyperdrive binding. Rejected for MVP scale; revisit when the admin surface grows beyond a handful of routes.
- **Admin endpoints on `api` behind Cloudflare Access and a dedicated `requireOperator()` middleware, no admin in the CLI (chosen).** One deploy, one app auth chain, one schema, with an edge gate in production. The operator boundary is a single middleware step that runs after Auth0 and before any **Scope** check. The admin routes live under a path prefix the CLI generator excludes, so they cannot leak into agent-facing tooling.

## Consequences

- **Admin routes live under `/admin/...` on `api`.** In production, Cloudflare Access protects the `/admin/...` path before the Worker handles the request. Every route under the prefix then runs `requireOperator()` immediately after Auth0 session resolution and before `requireScopes` (ADR 0034). Routes outside `/admin/...` ignore the operator flag.
- **`requireOperator()` middleware.**
  - Rejects requests that did not pass the Auth0 session check from ADR 0027 (Auth0 for **Workspace Member** authentication) — `not_found` envelope, no `unauthorized` distinction.
  - Loads the comma-separated `OPERATOR_EMAILS` Worker secret on cold start and compares against the Auth0 email claim with case-folded equality.
  - Rejects every **API Key** authentication attempt on `/admin/...` routes regardless of which **Scopes** the key holds. **API Keys** never reach `requireOperator()`.
  - Records the operator email on the request context so downstream handlers and **Audit Event** writers can use it as `actor.type = 'platform'`.
  - A non-operator session receives the generic `not_found` envelope to keep the admin surface non-enumerable from unauthorized identities.
- **`OPERATOR_EMAILS` is a Worker secret, not a database row.** Adding or removing an **Operator** is a `wrangler secret put` followed by a Worker redeploy. The allowlist is short (a handful of identities in the MVP) and rotating it through the same deploy path as other Worker secrets keeps the operator boundary outside any tenant-data path. The allowlist is not editable from the web admin surface itself; an operator cannot promote a new operator from the UI.
- **Web admin section on `app.agent-paste.sh`.** The same Auth0-authenticated SPA renders an additional `/admin` section conditionally on the email-allowlist check returning true. In production, Cloudflare Access also protects the `/admin` route. The section calls the `/admin/...` routes on `api`. Non-operators never see the section because the conditional render is gated on the same allowlist check the server enforces. Cross-site forgery is blocked by the existing same-site cookie configuration; admin requests carry the Auth0 session cookie and `Origin` is validated server-side.
- **CLI in `apps/cli` has no admin commands.** The OpenAPI spec generated from the **Zod schemas as source of truth for contracts** (ADR 0038) marks admin routes with `x-admin: true`, and the CLI code generator drops every route carrying that extension. The internal API client package (ADR 0037) likewise omits admin route bindings. There is no operator-only flag, no hidden command, no debug path. An operator who needs to act from the command line uses `curl` against the admin endpoint with their Auth0 session — and that is intentionally not ergonomic.
- **Scheduled rotation agent identity.** The scheduled remote agent that runs signing-key rotation per ADR 0045 authenticates through Cloudflare Access with a service token and through Auth0 Machine-to-Machine flow as a dedicated `rotation-agent@platform` operator email. It hits the same `/admin/...` rotation endpoint a human operator would. There is no separate cron token and no bypass path. The agent's M2M client is scoped to admin routes only; revoking the agent means revoking the Access service token, revoking the M2M client in Auth0, and dropping the email from `OPERATOR_EMAILS`.
- **Audit Events for operator actions.** Every admin route writes an **Audit Event** with `actor.type = 'platform'`, `actor.id` set to the operator email, and the action verb identifying the route. Operator actions are visible to **Workspace Members** of any affected **Workspace** through their existing **Audit Event** read path; the operator surface is observable to the tenants it affects.
- **Rate limits on `/admin/...`.** The Actor Rate Limit and Workspace Burst Cap from ADR 0039 do not apply, because the actor is `platform` and not a **Workspace Member** or **API Key**. A separate fixed cap of 60 requests per minute per operator email is enforced through the same Durable Object family; this is generous for human operators and prevents a compromised operator session from being weaponized into a high-volume abuse driver during the incident window before the credential is rotated.
- **Admin route surface in the MVP.** Initial routes: `POST /admin/lockdowns` (set), `DELETE /admin/lockdowns/:id` (lift), `GET /admin/lockdowns` (list), `POST /admin/rotations/:secretName` (on-demand storage-key rotation per ADR 0045), `GET /admin/audit/recent` (cross-workspace audit feed). Adding a new admin route is an explicit code change reviewed against the operator-boundary rule: the route must require `requireOperator()`, must never accept an **API Key**, must write an **Audit Event** with `actor.type = 'platform'`.
- **Cloudflare Access is required in production.** Access is an outer gate, not the source of platform authority. `requireOperator()` remains mandatory, **API Keys** remain rejected, and every operator action still writes an **Audit Event**. Local development and preview may disable Access when the route is not publicly reachable.
- **CONTEXT.md** adds **Operator** as a glossary term and pins the operator-boundary rules: operator is web-only, **API Keys** cannot assume operator authority, operator actions audit as `actor.type = 'platform'`.
