# Access Link Signed URL with Fragment-Encoded Payload

Status: Accepted. Supersedes the Access Link bearer-token portions of [ADR 0015](./0015-shared-auth-primitives-with-per-app-wiring.md), [ADR 0023](./0023-versioned-public-rest-apis.md), [ADR 0036](./0036-error-envelope-and-generic-404-boundary.md), [ADR 0043](./0043-bearer-credential-format-and-storage.md), and the old code-scoped Agent View discovery design replaced by [ADR 0052](./0052-agent-view-discovery-from-access-link-signed-urls.md).

An **Access Link** is no longer a stored bearer secret. It is a database row that can be minted into a self-validating signed URL on demand. The URL carries the signature in the **fragment** so it never reaches any server-side log surface. The URL shape is `https://app.agent-paste.sh/al/{publicId}#{base64url(payload)}` where `payload` is a hand-packed binary structure containing `(version, kid, exp, scopes_bitmask, sig)`. The signature is `HMAC_SHA_256(signing_key_kid, version || kid || exp || scopes || publicId_bytes)`. The `access_links` table holds `publicId`, `workspace_id`, `artifact_id`, allowed scopes, expiration window, and row lifecycle state — no secret, no ciphertext, no wrapping key. Per-artifact **Access Link Lockdown** is stored on the owning `artifacts` row and joined at resolve/mint time. This supersedes the `ap_al_{env}_{publicId}_{secret}` half of ADR 0043, eliminates the AES-256-GCM wrapping-key surface from ADR 0045, and removes the URL-borne credential leakage class from the platform.

## Considered Options

- **Bearer token in URL query or path (the original ADR 0043 design).** Simple and shareable. The token segment appears in Cloudflare Logpush, Axiom span attributes, Sentry breadcrumbs, Referer headers to any cross-origin asset the landing page loads, and the browser address bar. Every observability sink becomes a potential leak point, and the credential is forever-valid until rotated. Rejected: log-borne credential leakage is the dominant realistic threat for a URL-borne bearer.
- **Bearer token in URL fragment.** Fixes the log-leak class (fragments never reach servers) but keeps the credential model: the DB still holds a recoverable bearer, AES-256-GCM wrapping-key complexity stays, and a leaked fragment is forever-valid. A defensive improvement, not a structural one.
- **JWT or PASETO in fragment.** Brings standard tooling and the alg-confusion footgun family. The payload is twelve bytes of structured data; the standardization tax doesn't pay for itself when we sign and verify in the same Worker codebase.
- **JSON-then-base64url in fragment.** Easier to debug (operators can pipe the fragment through `jq`) at the cost of ~3× size. Defensible, but URL-level debugging is rare enough that the size tax doesn't justify itself.
- **Hand-packed binary payload, signed, fragment-borne (chosen).** Compact (~60 chars), fully opaque, self-validating, version-byte migration path, reuses the existing HMAC signing-key family from ADR 0028/0031 and the rotation playbook from ADR 0045.

## Consequences

- **URL shape**: `https://app.agent-paste.sh/al/{publicId}#{blob}`
  - Path segment is the 16-character Crockford base32 `publicId` (~80 bits). Safe to log; matches the existing `ap_*` redaction format.
  - Fragment is `base64url(payload_bytes)` and is never sent to any server.
  - Example: `https://app.agent-paste.sh/al/AB3CDEFGHJKLMN56#AQEAAAGJk2YAAAEC9XQrStUvWxYz0123456789AbCdEfGhIjKlMnOpQrStUvWxYz0`.
- **Payload layout** (44 bytes raw, 59 chars base64url):
  ```
  offset  size  field
  0       1     version       (current = 1)
  1       1     kid           (signing key generation, 1–255)
  2       8     exp           (big-endian uint64 epoch milliseconds)
  10      2     scopes        (big-endian uint16 bitmask of allowed read scopes)
  12      32    sig           (HMAC-SHA-256, 32 bytes)
  ```
- **Signature input** is the concatenation:
  `version (1B) || kid (1B) || exp (8B) || scopes (2B) || publicId_bytes (10B from Crockford base32 decode of the 16-char path segment)`
  Signed by the signing key identified by `kid`. The signing key is held as a Worker secret on `api` and rotates automatically on the 90-day cadence in ADR 0045, joining the content-gateway signing-key family from ADR 0028.
- **`access_links` table.** Columns: `id`, `workspace_id`, `artifact_id`, `revision_id` (for **Revision Links**), `public_id`, `type`, `scopes_bitmask`, `expires_at` (nullable; per-row absolute expiration, separate from per-URL `exp`), `created_at`, `created_by`, `revoked_at`. **No `secret`, no `secret_ciphertext`, no `secret_hmac`, no `wrap_kid`, no `hmac_kid`.** The signature is the credential.
- **Two expirations.** The row's `expires_at` is the durable end-of-life for the **Access Link** as a concept (the share is meant to die at that moment). The blob's `exp` is the URL's own time-bound, defaulted to `min(expires_at, now + 24h)` at mint time so a leaked URL ages out without operator action. A workspace member re-minting the URL gets a fresh `exp`; the row's `expires_at` is unchanged.
- **Mint path.** A **Workspace Member** or an **API Key** with read and share **Scopes** calls `POST /v1/access-links/:id/mint`. `api` loads the row (RLS enforces workspace scope per ADR 0044), confirms the row is active (not revoked, not expired, not under **Access Link Lockdown** or **Platform Lockdown**), picks the current signing-key `kid`, computes the signature, packs the binary payload, base64url-encodes it, and returns `{ url }`. Re-minting is idempotent semantically but produces a different URL each time (fresh `exp`).
- **Resolve path.** A browser opens the URL; the SPA landing page at `app.agent-paste.sh/al/{publicId}` reads `window.location.hash`, strips the leading `#`, base64url-decodes, and POSTs `{ publicId, blob }` to `POST /v1/access-links/resolve`. `api` runs the resolve under `withPlatformContext()` per ADR 0044 (the call is unauthenticated and cross-tenant by design): parse blob → reject on bad version → look up row by `publicId` and join Artifact and Platform Lockdown state → verify HMAC under `kid` (constant-time compare) → reject if `now > exp` → reject if `now > expires_at` → reject if **Access Link Lockdown** or **Platform Lockdown** is active → reject if `revoked_at` set → return **Agent View** plus signed content-gateway URLs per ADR 0028. Every failure path returns the same generic `not_found` envelope per ADR 0036 so callers cannot distinguish reasons.
- **Landing-page hardening.** `/al/{publicId}` is a minimal first-party route: no analytics, no third-party scripts, no external images or fonts, `Referrer-Policy: no-referrer`, and a route-specific CSP equivalent to `default-src 'none'; script-src 'self'; connect-src https://api.agent-paste.sh; style-src 'self'; base-uri 'none'; form-action 'none'; frame-ancestors 'none'`. Preview hosts use the matching preview API origin. This prevents the fragment credential from being handed to unrelated client code before it is posted to `api`.
- **Resolve-request logging.** The `blob` request body field is credential material once posted to `api`. Structured request logs, traces, errors, analytics events, and audit summaries must omit or redact it. CORS on the browser resolve endpoint is limited to the matching app origin; non-browser agents may call the same endpoint directly.
- **`api`-only resolution.** The `content` worker continues to authorize via the content-gateway signed token from ADR 0028. Access Links never resolve at `content`; they always go through `api` so the row's lockdown state and scopes can be enforced inside Postgres.
- **Recoverability changes shape, not behavior.** ADR 0015 says **Access Link** tokens are recoverable for authorized readers. Under the new model, "recovery" is re-minting on demand — the DB does not hold a token to return; the workspace re-signs `(publicId, exp, kid, scopes)` to produce an equivalent URL. UX is the same: an authorized member opens the link, copies the URL, shares it. Each retrieval mints a fresh-`exp` URL.
- **Revocation.** Setting `revoked_at` on the row makes every URL for that `publicId` fail at the row-state check, regardless of signature validity. No need to rotate signing keys to revoke a single link.
- **Access Link Lockdown** and **Platform Lockdown** are checked on the row at resolve time. The signed URL cannot bypass them.
- **Scopes bitmask.** The 16-bit bitmask in the payload pins what the URL is allowed to do at resolve time. The MVP uses `bit 0 = view artifact`, `bit 1 = list revisions`, `bit 2 = view raw content`. Future scopes (e.g., `bit 3 = download bundle`) consume unused bits. The bitmask is signed, so a sharer cannot escalate after the fact. Scopes broader than the row allows are rejected at resolve.
- **Mint volume cap.** The Actor Rate Limit and Workspace Burst Cap from ADR 0039 apply to mint calls. Resolve calls are unauthenticated and counted against the Artifact Rate Limit per ADR 0048.
- **Signing-key rotation.** The signing keys used here are the same HMAC-with-`kid` family as ADR 0028, with 90-day automatic rotation via the scheduled remote agent (ADR 0045). Old `kid`s remain valid for verify during the overlap window. A leaked **Access Link** URL becomes invalid at most `min(24h URL exp, signing-key overlap end)` after the rotation that drops its `kid`.
- **No client-side signature verification.** The SPA does not have the signing key. The browser issues a fetch with the blob; only `api` can verify.
- **No URL inspection without the secret.** A leaked URL discloses `publicId` (already loggable) and a 44-byte opaque blob. The blob fields are not encrypted, but reading them (e.g., learning the `exp` value) yields no information useful to an attacker without the signing key — and forging a URL with a tampered field fails HMAC verification.
- **Storage-key surface shrinks.** ADR 0045's storage-protection-key category now contains only the API Key pepper. The AES-256-GCM wrapping key is removed. The on-demand rotation playbook still exercises the pepper path on every PR per ADR 0045; the wrapping-key path is deleted along with the AES-GCM column.
- **No env crossing.** The signing key is environment-scoped (`production` and `preview` use different signing keys), so a URL minted in `preview` fails signature verification in `production`. The path subdomain (`app.agent-paste.sh` vs `app-preview.agent-paste.sh`) already segregates the resolve endpoints; the signature segregates them again at the crypto layer.
- **Migration impact**: zero. ADR 0043 was two days old when this ADR superseded the Access Link half, and no Access Link tokens have reached production. The `access_links` schema is built directly to this shape; no rewrite of existing rows is needed.
- **CONTEXT.md** adds **Access Link Signed URL** as a glossary term and updates **Bearer Credential Format** scope (now API Key only). Relationships pin the signed-URL shape, the fragment-encoded payload, the row-level revocation and lockdown rules, and the two-expirations model.
