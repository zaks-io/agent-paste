# Agent-First Ephemeral Publish and Write-Gated Monetization

Status: Accepted. Post-MVP phase; not a CLI-first MVP build gate per [ADR 0066](./0066-cli-first-mvp-contract-narrowing.md).

An agent can **Publish** with no human and no pre-existing credential by self-provisioning an **Ephemeral Workspace** behind a short-lived, low-cap **API Key**; reads are never the monetized axis because R2 egress is free; the gate that funds the service is the daily **write** allowance, and the conversion ladder is identity-as-trust: anonymous → claimed free account → `pro`. This realizes the "clean upgrade path to paid tiers" hook left open by [ADR 0048](./0048-transient-artifacts-by-default.md).

## Context

The platform is for agents, but every write path today requires an **API Key** minted by an authenticated **Workspace Member** through WorkOS ([ADR 0060](./0060-cli-authentication-via-auth0-loopback.md), [ADR 0068](./0068-workos-authkit-for-web-app-auth.md)). An unattended agent — a cloud OpenClaw run, a remote Cursor agent, an orchestration handing documentation between agents — cannot publish until a human completes a browser auth flow. That is the exact friction the product exists to remove, and it blocks the agent-to-agent handoff that is the core use case.

The naive fix — an open, unauthenticated write endpoint — collides with three load-bearing invariants. Every tenant table is RLS-scoped on `workspace_id` ([ADR 0044](./0044-workspace-isolation-via-postgres-rls.md)); the **Workspace** row has one writer behind one core ([ADR 0070](./0070-repository-core-ports-and-adapters.md)); and `CONTEXT.md` forbids "account" as a synonym for **Workspace**. An "**Artifact** with no **Workspace**" has no tenant to scope, no row to write, and no name in the domain.

Separately, the question of _what to charge for_ had to be settled before the credential model could be designed, because a rate limit is enforced against an identity and the identity model follows from the cost model. The stack's defining property is R2 with zero egress ([ADR 0005](./0005-cloudflare-workers-r2-postgres-hyperdrive.md)): a viral **Artifact** served to a large audience costs Worker invocations and R2 class-B ops measured in cents per million reads, not bandwidth. The marginal cost is therefore on the **write** (it creates durable rows, R2 objects, and storage liability), not the read. Gating reads would punish the audience, require per-**Artifact** claim machinery to "rescue" a popular link, and defend a cost that barely exists. The existing **Artifact Rate Limit** ([ADR 0048](./0048-transient-artifacts-by-default.md)) already caps read amplification as an abuse ceiling; nothing further is needed on the read axis.

Finally, an unauthenticated write surface that serves agent-authored HTML/JS is an abuse magnet — phishing, malware, illegal content, and SEO/link-farm spam — and the danger is domain-reputation poisoning, not cost. The defense must raise the cost of abuse without adding friction the honest agent notices.

## Decision

- **An agent self-provisions an Ephemeral Workspace, not "an account."** A new unauthenticated route mints a real, RLS-scoped **Workspace** owned by a reserved system actor, flagged ephemeral, with no **Workspace Member**. Every existing invariant holds unchanged: the **Artifact** has a `workspace_id`, RLS scopes it, the single **Repository** core ([ADR 0070](./0070-repository-core-ports-and-adapters.md)) writes it. "Anonymous" is a property of the tenant's claimed state, not the absence of a tenant. The credential it returns is an ordinary **API Key** in the `ap_pk_{env}_{publicId}_{secret}` shape ([ADR 0043](./0043-bearer-credential-format-and-storage.md)) with `write` and `read` **Scopes** only ([ADR 0061](./0061-mcp-worker-with-oauth-only-via-auth0-dcr.md)) — never `share` implicitly, never `admin`, which remains a **Member-Only Scope**.

- **Writes are the gate; reads stay free.** Monetization and abuse control both ride the daily new-**Artifact** write allowance, tiered by trust:

  - **Ephemeral (unclaimed):** a low per-source daily write cap, the shortest **Auto Deletion**, `noindex`, and the strictest scanning. Unraisable while unclaimed — anonymity _is_ the free trial.
  - **Claimed free (`free` Plan):** higher daily writes, normal **Auto Deletion**, persistence. The reason to sign up.
  - **`pro` Plan:** the daily write allowance lifted to a fair-use ceiling, plus the existing `pro` cap set ([ADR 0073](./0073-open-core-billing-plan-tiered-usage-policy-disabled-by-default.md)).
    A new **Artifact** counts against the allowance; updating an existing one (a new **Revision**) does not, bounded only by a generous per-**Artifact** lifetime **Revision** ceiling so a refinement loop cannot become a free-write firehose. Reads are not gated beyond the existing **Artifact Rate Limit** abuse ceiling.

- **Claim reparents the content into the claimer's existing Workspace.** An **Ephemeral Workspace** holds back a one-time **Claim Token**, returned only to the provisioning caller (the agent's own output), never embedded in any **Access Link Signed URL** ([ADR 0047](./0047-access-link-signed-url-with-fragment-encoded-payload.md)). A logged-in **Workspace Member** redeems it and the ephemeral tenant's **Artifacts** are reparented into that member's existing **Personal Workspace**, where they pick up the claimed `free` tier. Claim does not mint a second standalone tenant for the member to manage, because the product does not yet model multiple **Workspaces** per member; single-workspace-per-member is preserved and the source ephemeral tenant is marked consumed. The reparent is the single-core write ([ADR 0070](./0070-repository-core-ports-and-adapters.md)) and re-stamps `workspace_id`, so RLS ([ADR 0044](./0044-workspace-isolation-via-postgres-rls.md)) holds against the destination tenant. A viral public link carries no claim power, so possession of the shared URL grants no ownership. No **Claim Token** retained means the content is genuinely ownerless and simply reaches **Auto Deletion**; the cost of pure anonymity is that it cannot be promoted, only re-published.

- **Proof-of-work, not CAPTCHA, gates the ephemeral write.** The agent solves a hashcash-style challenge issued by the Worker before the write is accepted — headless-solvable in roughly a few hundred milliseconds, and ruinous at volume because cost scales with write count. Cloudflare Turnstile is rejected on the agent path precisely because it is a browser challenge a headless agent cannot solve; using it would block the hero use case. Turnstile and the in-Worker bot score are browser/Enterprise concerns reserved for the human claim and upgrade surfaces, never the agent write.

- **Anti-abuse is layered and asymmetric — invisible to honest agents, expensive at volume.** In priority order: (1) the isolated **Content Origin** plus strict **Execution Policy** already contain hostile JS ([ADR 0030](./0030-mvp-execution-policy-cdn-allowlisted-csp.md), [ADR 0001](./0001-private-artifact-storage-behind-controlled-origin.md)); this is the architectural prerequisite and it already exists. (2) Aggressively short **Auto Deletion** for ephemeral content limits how long bad content lives and how worthwhile abuse is. (3) Native `[[ratelimits]]` bindings ([ADR 0064](./0064-native-ratelimit-bindings-for-authenticated-counters.md)) dampen the write rate per source, with a Durable Object counter for the few caps that need a strongly consistent global ceiling, since the binding is per-PoP and eventually consistent. (4) The **Safety Scanner** seam ([ADR 0051](./0051-safety-scanner-lifecycle.md)) is where stronger moderation plugs in for ephemeral content under its own `scanner_id`: Workers AI Llama Guard 3 on submitted text at scan time, and an async Cloudflare URL Scanner verdict on the published URL, both advisory — a malicious verdict drives **Platform Lockdown** ([ADR 0040](./0040-platform-lockdown-for-operator-initiated-takedown.md)), no schema churn. (5) `noindex`/`nofollow` on ephemeral content removes SEO-farm value. The native bot score and JA3/JA4 fingerprinting are Enterprise-only and deliberately not adopted pre-scale.

- **Script execution requires a claimed tier, enforced at serve time, not by content inspection.** Executable JavaScript is the high-risk class — sandboxed but still able to drive an exfiltrating fake login — and the platform will only execute agent-authored code behind an attributable, claimable identity with an audit trail. The ephemeral tier serves **Untrusted Content** under a second, script-disabled **Execution Policy** (`script-src 'none'`, no inline, no event handlers), so inert content (markdown, text, JSON, images) and static HTML/CSS render identically while any embedded script simply never runs. This refines [ADR 0030](./0030-mvp-execution-policy-cdn-allowlisted-csp.md)'s single fixed policy by introducing a tier-selected variant, and it fails closed: the line is enforced by the sandbox, not by parsing HTML for `<script>`, so unknown smuggling vectors (SVG `<script>`, `srcdoc`, data-URI script) also do not execute. A publish-time detector may still raise an advisory **Safety Warning** ([ADR 0051](./0051-safety-scanner-lifecycle.md)) noting that script is present and dormant until the tenant is claimed, as a conversion nudge — but that detector is never the security control. Claiming the **Workspace** promotes it to the normal CDN-allowlisted **Execution Policy** and script runs.
- **`content` selects the Execution Policy from the content-gateway token, not the database.** `content` has no Hyperdrive binding ([ADR 0028](./0028-signed-url-tokens-for-content-gateway-authorization.md)) and must not gain one. The tier signal rides in the signed content-gateway token payload: `api` sets a script-disabled bit at mint time for ephemeral tenants, and `content` picks the script-disabled CSP from the verified token with no lookup. An absent or unverifiable bit fails closed to the script-disabled policy.
- **The identity ladder is the trust ladder.** More identity means looser limits and softer scanning because there is more to lose and someone to hold accountable: an **Ephemeral Workspace** has nothing, a claimed `free` **Workspace** has a WorkOS email to ban, and a `pro` **Workspace** has a payment method on file — the strongest anti-abuse signal there is. The tiers price trust, not just capacity, and channel everyone who wants more into a tier where consequences are enforceable. Script execution is the sharpest rung: it requires a claimed tenant precisely so there is an audit trail behind any code the platform runs.

## Surfaces and Trust Boundaries

- **`api`** hosts two new unauthenticated routes through the [ADR 0072](./0072-contract-driven-route-registrar-and-guard.md) registrar: a proof-of-work-gated ephemeral-provision endpoint that mints the **Ephemeral Workspace** + **API Key** + **Claim Token**, and a claim-redemption endpoint that requires a `workos_access_token` (it is a human action). Both run under a reserved system actor through `runCommand` ([ADR 0035](./0035-runcommand-sequencing-and-idempotency-records.md)); provisioning and claiming emit **Audit Events**. The ephemeral write itself reuses the existing authenticated **Upload Session** and **Publish** paths once the minted **API Key** is in hand — no new write surface, just a new way to obtain the key.
- **`upload`** and **`content`** are unchanged. `content` stays DB-free ([ADR 0028](./0028-signed-url-tokens-for-content-gateway-authorization.md)); the read-side **Artifact Rate Limit** remains KV/binding-only on `content` and is the only read gate. Ephemeral status rides on the **Artifact**/`Workspace` rows the authenticated paths already load, so no new `content` lookup is introduced.
- **`jobs`** gains the ephemeral-tier scan routing under the existing **Safety Scanner** consumer and honors the shorter ephemeral **Auto Deletion** through the existing sweep ([ADR 0048](./0048-transient-artifacts-by-default.md)); no new cron topology ([ADR 0032](./0032-jobs-worker-trigger-model-and-queue-topology.md)).
- **Turnstile** appears only on `apps/web` claim/upgrade surfaces, never on an agent-facing route.

## Considered Options

- **Fully anonymous, accountless publish returning only a signed URL.** Cleanest for one-shot agent-to-agent handoff, but breaks RLS tenancy, leaves nothing to rate-limit a publisher against, and cannot be promoted to a paid relationship. The **Ephemeral Workspace** keeps the frictionless feel while preserving every tenancy invariant and the upgrade path.
- **Gate reads instead of writes.** Matches the intuition that a viral link is "expensive," but R2 zero-egress makes reads nearly free, and per-**Artifact** read gating punishes the audience and forces claim machinery to rescue a popular link whose owner is a ghost. Rejected; the existing **Artifact Rate Limit** is sufficient abuse defense on reads.
- **No limits at all ("fuck it").** Correct for reads and adopted there. Rejected for writes: an unauthenticated write endpoint with no ceiling is a free-hosting firehose and a row-exhaustion vector, independent of egress cost.
- **Turnstile / Cloudflare Bot Management on the write path.** The platform's own anti-bot stack cannot distinguish the product's hero user (an automated client) from an abuse bot, and the in-Worker bot score is Enterprise-only. Rejected on the agent path; proof-of-work is the agent-friendly equivalent and Turnstile is reserved for human surfaces.
- **Model the agent as a Workspace Member.** A member is pinned to a `workos_user_id` an agent does not have, and `admin` is a **Member-Only Scope**. Rejected; the agent holds an **API Key** against an unclaimed tenant, consistent with [ADR 0043](./0043-bearer-credential-format-and-storage.md) and [ADR 0061](./0061-mcp-worker-with-oauth-only-via-auth0-dcr.md).

## Consequences

- An agent publishes in one call with no human in the loop, and the resulting link works immediately; the agent-to-agent handoff use case is unblocked without weakening the authenticated path.
- "Anonymous" is implemented as an unclaimed **Ephemeral Workspace**, so RLS ([ADR 0044](./0044-workspace-isolation-via-postgres-rls.md)), the single-core writer ([ADR 0070](./0070-repository-core-ports-and-adapters.md)), and audit ([ADR 0004](./0004-audit-state-changes-through-wrapper.md)) are unchanged. Ephemeral and claimed tenants differ by a flag and a cap set, not by code path.
- Monetization sells durability and write volume, not reach. The upgrade pitch is "keep your thing alive and publish more," which users do not resent, rather than "pay so others can see it," which they do.
- The new `pro` revenue driver is the lifted daily write allowance, layered onto the [ADR 0073](./0073-open-core-billing-plan-tiered-usage-policy-disabled-by-default.md) tier selection within the [ADR 0056](./0056-mvp-usage-policy-defaults-and-platform-caps.md) ceilings. The concrete daily-write numbers per tier are platform-controlled values defined in the usage-policy ledger, not hardcoded here.
- Reads remain free and ungated beyond the existing **Artifact Rate Limit**, so the viral-link cost stays a rounding error and no claim machinery is needed to keep a popular link serving.
- Stronger moderation (Llama Guard 3, URL Scanner) enters through the existing **Safety Scanner** interface as a code-only change under a new `scanner_id` ([ADR 0051](./0051-safety-scanner-lifecycle.md)); no schema migration, and warnings/lockdowns flow through paths that already exist.
- Executable HTML is not served anonymously: the ephemeral tier runs a script-disabled **Execution Policy**, so the in-iframe-phishing risk is reduced to _static_ visual mimicry (a page that looks like a login but cannot act), further bounded by short ephemeral **Auto Deletion**, async URL Scanner verdict, `noindex`, and fast **Platform Lockdown**. Any content the platform actually executes sits behind a claimed, auditable tenant. The cost is that the one-line anonymous publish of an interactive HTML/JS simulation requires a (free) claim first; the agent-to-agent handoff and static-page share stay fully anonymous.
- Introducing a second **Execution Policy** means `content` now branches its response CSP on a token-carried bit, and the policy set is no longer literally "one fixed policy" ([ADR 0030](./0030-mvp-execution-policy-cdn-allowlisted-csp.md), noted there). The branch is small and the script-disabled variant is the safe default, so the added surface is bounded.
- Proof-of-work is hand-rolled because Cloudflare exposes no developer PoW primitive; it is a small, self-contained challenge/verify the `api` Worker owns.
